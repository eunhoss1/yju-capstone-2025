2. 수익 모델의 부재: "돈이 안 된다"

"필요는 하지만 돈이 안 돼서 안 만들었다"는 것이 냉정한 결론입니다. 수요는 있지만, 아무도 지불할 의사가 없는 시장 구조입니다.

기사 (개인사업자)

→ "SaaS 구독? 안 사."

중소 배송대리점

→ "비용 절감 압박 심해. 안 사."

대기업 본사

→ "자체 시스템 있어. 안 사."

기술은 가능하나 사업이 애매하니, 스타트업도 투자를 안 합니다.

그래서, 이게 왜 '의미 있는' 졸업작품인가?

기업은 수익성 문제로 쉽게 못 건드린 기술을,
우리는 '학술적 관점'에서 자유롭게 만들 수 있는 유일한 위치에 있기 때문입니다.

기업 (Enterprise)

  • 수익성(ROI) 계산해야 함
  • 개발비/유지보수비 발생
  • 정치적/이해관계 복잡
  • → "투자 우선순위에서 제외."

우리 (졸업작품)

  • 수익성(ROI) 필요 없음
  • 개발비/유지보수비 0원
  • 정치/이해관계 없음
  • → "순수 기술 관점에서 구현 가능."

4. 기술적 현실성: 100% 구현 가능 (우수 주제)

이 프로젝트는 "통제 불가능한 변수"가 0입니다. 모든 기술은 이미 존재하며, 우리는 '조합'과 '연결'에 집중합니다. 난이도 조절(Scoping)이 완벽하게 가능합니다.

① 경로 최적화 (API)

우리가 '발명'할 필요가 없습니다. TMAP, Naver 등이 이미 완벽하게 해결해 놨습니다.

우리의 임무 (난이도: 하)

API '주문서(Request)'에 주소 엑셀을 던지고, API '답변서(Response)'로 오는 경로 순서(A→C→B)를 받기만 하면 됩니다.

② AI 적재 (알고리즘)

'완벽한 3D 테트리스 신(神)'을 만드는 게 아닙니다. '현실적인 룰(Rule)'을 따르는 똑똑한 알고리즘을 만듭니다.

[Easy Mode] 1년 목표 (난이도: 중상)

'if문'과 '정렬(Sort)'로 4가지 현실 룰(P0~P3: 묶음, 무게, LIFO, 공간)을 구현, 'Good-enough'한 (x,y,z) 좌표를 계산합니다.

[Hard Mode] 미래 비전

유전 알고리즘 등을 도입해 빈 공간 0.1%까지 줄이는 '완벽한 최적화'에 도전합니다.

③ 3D 시각화 (Three.js)

'3D 게임 엔진'을 만드는 게 아닙니다. '좌표값 뷰어'를 만드는 겁니다.

[Easy Mode] 1년 목표 (난이도: 중)

백엔드가 준 (x,y,z) 좌표대로, 튜토리얼 예제를 활용해 '회색 박스'를 3D 공간에 `for`문으로 그려줍니다.

[Hard Mode] 미래 비전

트럭 모델링을 입히고, 박스 스티커를 붙이고, 적재 애니메이션을 추가합니다.

5. 최종 비전: '뇌(S/W)'와 '몸(H/W)'의 결합

우리는 1년 동안 '뇌(소프트웨어)'를 완벽하게 만듭니다. 이 '뇌'는 '몸(로봇)'으로 즉시 확장될 수 있습니다.

Phase 1: '뇌' (S/W) - (우리의 1년 목표)

AI가 계산한 (x,y,z) 좌표 설계도를 '웹사이트(3D 뷰어)'로 시각화합니다.

→ "사람 기사님"이 3D 뷰를 보고 '직접' 적재 (설명서)
+

Phase 2: '몸' (H/W) - (미래 비전)

AI가 계산한 (x,y,z) 좌표 설계도를 '자동화 로봇'으로 전송합니다.

→ "로봇 팔"이 설계도를 보고 '자동' 적재 (명령서)

결론: 가치 있는 "졸업작품 주제"

기업은 '수익성' 때문에 안 만든 기술을, 학생은 '기술적 가치'로 만들 수 있습니다.
수요는 존재하나, 시장 구조의 문제로 상용화되지 않은 것.
이것이 우리 프로젝트가 '의미 있는 틈새'인 이유입니다.
